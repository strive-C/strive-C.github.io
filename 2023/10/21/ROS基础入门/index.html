ROS 介绍

  • ROS = 通信机制 + 开发工具 + 应用功能 + 生态系统

    • 通信机制：松耦合分布式通信
    • 开发工具：命令行、TF 坐标变换、Gazebo、Rviz、QT 工具箱
    • 应用功能：很多
    • 生态系统：
      • 发行版
      • 软件源
      • ROS wiki：记录 ROS 信息文档的主要论坛
      • 邮件列表
      • ROS Answers
      • 博客：http://www.ros.org/news

  • 节点 Node 和 节点管理器 ROS Master

    • 节点：执行单元
      • 执行具体任务的进行、独立运行的可执行文件
      • 不同节点可使用不同的编程语言，可分布式运行在不同主机中
      • 节点在系统中的名称必须是唯一的
    • 节点管理器：控制中心
      • 为节点提供命名和注册服务
      • 跟踪和记录 话题 / 服务 通信，辅助结点相互查找、建立连接
      • 提供参数服务器，节点使用此服务器存储和检索运行时的参数

  • 话题 Topic：异步通信机制

    • 节点间用来传输数据的重要总线
    • 使用发布/订阅模式，数据由发布者传输到订阅者，同一个话题的订阅者或发布者可以不唯一

  • 消息 Message：话题数据

    • 具有一定的类型和数据结构，包括 ROS 提供的标准类型和用户自定义类型
    • 使用编程语言无关的.msg文件定义，编译过程中生成对应的代码文件

  • 服务 Service：同步通信机制

    • 使用客户端/服务器模式，客户端发送请求数据，服务器完成处理后返回应答数据
    • 使用编程语言无关的.srv文件定义请求和应答数据结构，编译过程中生成对应的代码文件
  • 话题 服务
    同步性 异步 同步
    通信模型 发布/订阅 客户端/服务器
    底层协议 ROSTCP/ROSUDP ROSTCP/ROSUDP
    反馈机制 无 有
    缓冲区 有 无
    实时性 弱 强
    节点关系 多对多 一对多
    使用场景 数据传输 逻辑处理

  • 参数 Parameter：全局共享字典

    • 可通过网络访问的共享、多变量字典
    • 节点使用此服务器来存储和检索运行时的参数
    • 适合存储静态、非二进制的配置参数，不适合存储动态配置的参数

  • 功能包 Package

    • ROS 软件中的基本单元，包含节点源码、配置文件、数据定义等

  • 功能包清单 Package manifest

    • 记录功能包的基本信息，包含作者信息、许可信息、依赖选项、编译标志等

  • 元功能包 Meta Packages

    • 组织多个用于同一目的功能包

  • 在package.xml中添加功能包依赖

    • <build_depend>message_generation</build_depend>：这一行指定了在构建（编译）该ROS软件包时所需的依赖项。具体来说，它表示该软件包依赖于名为 message_generation 的软件包，用于生成ROS消息（Message）的包。在构建时，软件包将使用 message_generation 中的工具来处理和生成消息，以确保软件包正确地与其他ROS节点通信。

    • <exec_depend>message_runtime</exec_depend>：这一行指定了在运行时（当软件包被执行时）所需的依赖项。它表示该软件包在运行时依赖于名为 message_runtime 的软件包。message_runtime 包含运行时库，这些库允许软件包在运行时解析和使用ROS消息。

    • 这些依赖关系的明确定义有助于ROS工具正确构建和部署软件包，以确保它们在运行时具有所需的依赖项。当构建和运行ROS软件包时，ROS工具（如catkin_make和rosrun）将自动处理这些依赖关系，以确保软件包能够正常工作。

  • 在功能包中的CMakeLists.txt中添加编译选项

    • 在find_package中添加功能包message_generation

    • add_message_files(FILES Person.msg)：这一行代码告诉ROS构建系统，要将名为 Person.msg 的消息文件添加到该软件包中。这是一种将自定义消息定义添加到ROS软件包的方式。Person.msg文件包含了消息的结构和字段定义。当你运行catkin_make等构建工具时，ROS构建系统会处理这些消息定义文件，并生成相应的消息代码，以便在ROS程序中使用。

    • generate_messages(DEPENDENCIES std_msgs)：这一行代码告诉ROS构建系统在生成消息时依赖于其他消息包。在这种情况下，它依赖于std_msgs软件包，这是ROS中包含许多标准消息类型的软件包。这意味着，如果你的Person.msg消息定义中使用了std_msgs中定义的消息类型，你需要指定这个依赖项，以确保生成的消息代码能够正确引用 std_msgs 中的消息类型。

launch 启动文件

介绍

  • Launch 文件：通过 XML 文件实现多节点的配置和启动（可自动启动 ROS Master）

  • launch 文件：

    • <launch>为文件的根元素
    • <node>为启动节点
      • pkg：节点所在功能包名称
      • type：节点的可执行文件名称
      • name：节点运行时的名称
      • output：控制节点是否要将控制信息打印到当前终端
      • respawn：控制节点挂掉后，是否进行重启
      • required：是否在launch文件，该结点一定要启动起来
      • ns：namespace 命名空间
      • args：结点的输入参数
      • 前三个属性是必须属性，后面的属性为可选属性
    • <param name="output_frame" value="odom" />
      • 设置 ROS 系统运行中的参数，存储在参数服务器中
      • name：参数名
      • value：参数值
    • <rosparam file="params.yaml" command="load" ns="params" />
      • 加载参数文件中多个参数
    • <arg name="arg-name" default="arg-value" />
      • launch 文件内部的局部变量，仅限于 launch 文件使用
      • name：参数名
      • value：参数值
      • 调用方法：
        • <param name="foo" value="$(arg arg-name)" />
        • <node name="node" pkg="package" type="type" args="$(arg arg-name)" />
    • <remap from="/turtlebot/cmd_vel" to="/cmd_vel" />
      • 重映射 ROS 计算图资源的命名
      • from：原命名
      • to：映射之后的命名
    • <include file="$(dirname)/other.launch" />
      • 包含其他 launch 文件，类似 C 语言中的头文件包含
      • file：包含的其他 launc 文件路径
      • 包含的文件会运行一遍
    • 其他详见：https://wiki.ros.org/roslaunch/xml
